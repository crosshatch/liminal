# Cloudflare Assets

`Assets` is the `liminal` wrapper around Cloudflare's static asset binding.

It gives you two things:

- `Assets.layer(binding)` to validate and provide the binding as an Effect service
- `Assets.forward` to proxy the current `NativeRequest` to the asset binding

## Provide the Assets binding

The Cloudflare env binding name is supplied when you build the layer:

```ts
import { Assets } from "liminal"

const AssetsLive = Assets.layer("ASSETS")
```

The convention is to use `ASSETS` as the binding name, but anything that matches your Wrangler configuration works.

## Use `Assets.forward` in a catch-all route

A typical pattern forwards unmatched requests to static assets:

```ts
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"
import { Assets } from "liminal"

export const ApiLive = Layer.mergeAll(HttpRouter.add("*", "/*", Assets.forward))
```

You can also add response headers before forwarding:

```ts
import { Effect, Layer } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"
import { Assets } from "liminal"

export const ApiLive = Layer.mergeAll(
  HttpRouter.add(
    "*",
    "/*",
    Assets.forward.pipe(Effect.map(HttpServerResponse.setHeader("Cache-Control", "public, max-age=3600"))),
  ),
)
```

For either pattern to run, `Assets.layer("ASSETS")` needs to be included in the Worker runtime alongside the rest of
your bindings.

## Access the raw binding

For lower-level behavior than `forward`, yield the binding directly:

```ts
import { Effect } from "effect"
import { Assets } from "liminal"

Effect.gen(function* () {
  // ...

  const assets = yield* Assets.Assets
  const response = yield* Effect.promise(() => assets.fetch(request))
})
```
