# Cloudflare R2

`R2` is the `liminal` wrapper around a Cloudflare R2 bucket binding.

It exposes the bucket as a typed Effect service. The wrapper is intentionally thin — call the standard Cloudflare R2
API on the underlying bucket via `Effect.promise(...)`.

## Provide the R2 binding

```ts
import { R2 } from "liminal"

const BucketLive = R2.layer("BUCKET")
```

The string passed to `.layer(...)` must match the `binding` field in your Wrangler config.

## Read and write objects

```ts
import { Effect } from "effect"
import { R2 } from "liminal"

const putAvatar = (key: string, body: ArrayBuffer) =>
  Effect.gen(function* () {
    const bucket = yield* R2.R2
    yield* Effect.promise(() => bucket.put(key, body))
  })

const getAvatar = (key: string) =>
  Effect.gen(function* () {
    const bucket = yield* R2.R2
    const object = yield* Effect.promise(() => bucket.get(key))
    if (object === null) return undefined
    return yield* Effect.promise(() => object.arrayBuffer())
  })
```

## Use it as the backing store for `KeyValueStore`

For most actor storage needs, you do not want to manipulate R2 directly — use Effect Platform's `KeyValueStore`
abstraction backed by R2:

```ts
import { Layer } from "effect"
import { R2 } from "liminal"
import { KeyValueStore } from "liminal/platform"

export const KvLive = KeyValueStore.layerR2().pipe(Layer.provide(R2.layer("BUCKET")))
```

See [KeyValueStore on R2](./key-value-store.mdx) for full details.

## Wrangler configuration

```jsonc
{
  "r2_buckets": [
    {
      "binding": "BUCKET",
      "bucket_name": "my-bucket",
    },
  ],
}
```

The `binding` field must match the name you pass to `R2.layer(binding)`.
