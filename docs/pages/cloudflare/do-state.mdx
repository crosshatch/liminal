# Durable Object State

`DoState` exposes the current Durable Object's `DurableObjectState` as a typed Effect service.

You only need it inside Durable-Object-side code — that is, anything that runs through an `ActorRegistry`. `Worker.make`
handlers do not have a `DoState` and should not depend on one.

## Available inside actor registries

`ActorRegistry` provides `DoState` automatically while running handlers, `onConnect`, and any layer derived from the
registry's `prelude` or `runLayer`.

```ts
import { Effect } from "effect"
import { DoState } from "liminal"

Effect.gen(function* () {
  const state = yield* DoState.DoState

  const value = yield* Effect.promise(() => state.storage.get("counter"))
})
```

Common uses:

- read or write durable storage via `state.storage.get/put/delete/list`
- block on an alarm via `state.storage.setAlarm(...)`
- enumerate accepted WebSockets via `state.getWebSockets()`
- block concurrent operations via `state.blockConcurrencyWhile(...)`

## Persist arbitrary data with `state.storage`

Durable Object storage is a small key-value store private to the actor instance. Use it for short, low-frequency state
that should survive hibernation but does not warrant R2 or D1.

```ts
Effect.gen(function* () {
  const state = yield* DoState.DoState

  yield* Effect.promise(() => state.storage.put("counter", 42))
})
```

For larger data, prefer one of:

- [R2](./r2.mdx) for blobs
- [D1](./d1.mdx) for relational data
- [KV](./kv.mdx) for cross-instance configuration
- [`KeyValueStore.layerR2`](./key-value-store.mdx) for the Effect Platform `KeyValueStore` interface backed by R2
