# KeyValueStore on R2

`KeyValueStore.layerR2(...)` (from `liminal/platform`) provides the Effect Platform `KeyValueStore` interface, backed by
a Cloudflare R2 bucket.

This is the recommended way to give actor handlers access to durable storage when you do not need the structure of a
relational database — every read and write goes through R2 transparently.

## Build the layer

```ts
import { Layer } from "effect"
import { R2 } from "liminal"
import { KeyValueStore } from "liminal/platform"

export const KvLive = KeyValueStore.layerR2().pipe(Layer.provide(R2.layer("BUCKET")))
```

`layerR2()` produces a layer that depends on `R2`, so you provide `R2.layer("BUCKET")` underneath it.

## Optional `root` prefix

Pass `root` to namespace all keys behind a path prefix:

```ts
import { KeyValueStore } from "liminal/platform"

KeyValueStore.layerR2({ root: "games" })
```

The trailing slash is added for you, so a key `"abc"` becomes `"games/abc"` in R2. This is useful when one bucket hosts
multiple logical stores.

## Use the standard Effect Platform API

Once provided, you yield `KeyValueStore.KeyValueStore` and use the standard Effect Platform interface:

```ts
import { Effect } from "effect"
import { KeyValueStore } from "effect/unstable/persistence"

const writeBoard = (id: string, encoded: string) =>
  Effect.gen(function* () {
    const kv = yield* KeyValueStore.KeyValueStore
    yield* kv.set(id, encoded)
  })

const readBoard = (id: string) =>
  Effect.gen(function* () {
    const kv = yield* KeyValueStore.KeyValueStore
    return yield* kv.get(id)
  })
```

Available operations: `get`, `getUint8Array`, `set`, `remove`, `clear`, `size`. Errors surface as
`KeyValueStore.KeyValueStoreError` so failures stay distinguishable from your domain errors.

## When to choose this over `Kv`

- Use [`Kv`](./kv.mdx) when you have small values and want Cloudflare KV's eventual-consistency edge cache.
- Use this `KeyValueStore.layerR2` when you want the Effect Platform `KeyValueStore` interface — for example so the same
  code can run against a filesystem-backed store in tests.
- Use the raw [`R2`](./r2.mdx) binding when you need streaming, multipart uploads, or other R2-specific features.

## Use it from inside an actor registry

Wire the layer through `prelude` so handlers can yield the store:

```ts
import { Layer } from "effect"
import { ActorRegistry, R2 } from "liminal"
import { KeyValueStore } from "liminal/platform"

import { ChatActor } from "./ChatActor.ts"
import * as handlers from "./handlers.ts"
import { onConnect } from "./onConnect.ts"

const KvLive = KeyValueStore.layerR2().pipe(Layer.provide(R2.layer("BUCKET")))

export class ChatRegistry extends ActorRegistry.Service<ChatRegistry>()("ChatRegistry", {
  actor: ChatActor,
  prelude: KvLive,
  runLayer: Layer.empty,
  onConnect,
  handlers,
}) {}
```
