# Cloudflare Hyperdrive

`Hyperdrive` is the `liminal` wrapper around a Cloudflare Hyperdrive binding. It exposes the binding as a typed Effect
service and surfaces the connection string as a `Redacted<string>`.

## Provide the Hyperdrive binding

```ts
import { Hyperdrive } from "liminal"

const HyperdriveLive = Hyperdrive.layer("HYPERDRIVE")
```

## Read the connection string

```ts
import { Hyperdrive } from "liminal"

Hyperdrive.connectionString
```

## Local dev fallback

Use Hyperdrive in production, but swap to a local Postgres URL in development:

```ts
import { Effect, Redacted } from "effect"
import { Hyperdrive } from "liminal"

import { AppConfig } from "./AppConfig.ts"

export const connectionString = Effect.flatMap(AppConfig, ({ dev }) =>
  dev ? Effect.succeed(Redacted.make("postgres://dev:dev@localhost:5432/myapp")) : Hyperdrive.connectionString,
)
```

## Wrangler configuration

Your Wrangler config must declare the Hyperdrive binding:

```jsonc
{
  "hyperdrive": [
    {
      "binding": "HYPERDRIVE",
      "id": "<your-hyperdrive-config-id>",
    },
  ],
}
```

The `binding` field here must match the name you pass to `Hyperdrive.layer(binding)`.
