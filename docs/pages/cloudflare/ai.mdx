# Workers AI

`Ai` is the `liminal` wrapper around the Cloudflare Workers AI binding.

It exposes the binding as a typed Effect service so model invocations can be threaded through the same layer wiring as
the rest of your dependencies.

## Provide the AI binding

```ts
import { Ai } from "liminal"

const AiLive = Ai.layer("AI")
```

The convention is to use `AI` as the binding name, but anything that matches your Wrangler configuration works.

## Run a model

The wrapper is intentionally thin — yield the service and call `run` directly:

```ts
import { Effect } from "effect"
import { Ai } from "liminal"

const generate = Effect.gen(function* () {
  const ai = yield* Ai.Ai

  const result = yield* Effect.promise(() =>
    ai.run("@cf/meta/llama-3-8b-instruct", {
      prompt: "Write a haiku about durable objects.",
    }),
  )

  return result
})
```

The `ai.run(...)` signature is the standard Cloudflare Workers AI signature; refer to the
[Workers AI docs](https://developers.cloudflare.com/workers-ai/) for available models and inputs.

## Wrangler configuration

```jsonc
{
  "ai": {
    "binding": "AI",
  },
}
```

The `binding` field must match the name you pass to `Ai.layer(binding)`.
