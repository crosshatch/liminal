# Accumulator

An `Accumulator` is a typed state container driven by events. It reduces incoming events into long-lived client state,
giving UI code a stable state stream instead of forcing every component to understand the full event log.

If you are building anything larger than a toy screen, this is the most important state guide after the quickstart.

## Define an accumulator

An accumulator is a tagged service parameterized by its state type.

```ts
import { Stream } from "effect"
import { Accumulator } from "liminal"

import { TicTacToeClient, Player } from "./TicTacToeClient.ts"

type Item = Stream.Success<typeof TicTacToeClient.events>

export class GameState extends Accumulator.Service<
  GameState,
  {
    readonly name: typeof Player.Type
  }
>()("examples/GameState") {}
```

The state is a plain TypeScript type. There is no schema config; you control the shape directly.

## Extract initial state

The `initial` function inspects each event until it finds one that produces the starting state.

```ts
import { Effect, Option } from "effect"

const initial = Effect.fn(function* (item: Item) {
  if (item._tag === "GameStarted") {
    return Option.some({ name: item.player })
  }
  return Option.none()
})
```

Until `initial` returns `Some`, the accumulator is not active and no reductions occur.

## Reduce events

The `reduce` function folds each subsequent event into the current state.

```ts
const reduce = (event: Item) => (state: { readonly name: typeof Player.Type }) => {
  switch (event._tag) {
    case "MoveMade":
      return Effect.succeed(state)
    case "GameEnded":
      return Effect.succeed(state)
    default:
      return Effect.succeed(state)
  }
}
```

## Use typed per-event reducers

The `reducer` helper narrows the event type to a single tag, which is useful when composing reducers.

```ts
const reducer = GameState.reducer<Item>()

const GameStarted = reducer(
  "GameStarted",
  ({ player }) =>
    () =>
      Effect.succeed({ name: player }),
)

const MoveMade = reducer("MoveMade", () => (state) => Effect.succeed(state))

const GameEnded = reducer("GameEnded", () => (state) => Effect.succeed(state))
```

Each call returns the reducer function unchanged. The value is in the type narrowing, not runtime behavior.

## Build the layer

Combine the source stream, initial function, and reduce functions into a layer.

```ts
export const layer = GameState.layer({
  source: TicTacToeClient.events,
  initial,
  reduce: Match.valueTags({ GameStarted, MoveMade, GameEnded }),
})
```

That layer starts the reduction loop and exposes:

- `GameState.get`: an `Effect` that reads the current state
- `GameState.stream`: a `Stream` of state changes

## Consume state

```ts
Effect.gen(function* () {
  const state = yield* GameState.get
  const stateStream = GameState.stream
})
```

Atoms can subscribe to the stream directly:

```ts
import { Atom } from "effect/unstable/reactivity"

import { runtime } from "./runtime"
import { GameState } from "./State"

export const stateAtom = runtime.atom(GameState.stream)
```

## When to use it

Use `Accumulator` when:

- multiple screens care about the same evolving state
- reconnects should replay state from events
- `onConnect` already emits a snapshot event
- you want a stable state stream derived from events

Skip it when a screen only needs one or two direct method calls and no long-lived local state.

Read [Snapshot and Delta Events](/state/snapshot-delta-events) for the event pattern that makes accumulators reliable.
