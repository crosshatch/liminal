# Quickstart: Cloudflare

Let's cover the simplest complete path through `liminal`.

We'll implement a trimmed-down Tic-Tac-Toe example:

- Define a typed client (the public protocol).
- Define an actor with per-client attachments.
- Implement one handler and one `onConnect` hook.
- Register the actor within a Cloudflare Durable Object namespace.
- Upgrade the client from an HTTP route.
- Consume the client from a browser with `Client.layerSocket(...)`.

For larger apps, split methods, events, handlers, and lifecycle files into separate modules. The next guides cover that
structure in more detail.

## Define the client

Start with the protocol the browser and actor will share.

```ts
// TicTacToeClient.ts
import { Schema as S } from "effect"
import { Client } from "liminal/actor"

export const Player = S.Literals(["X", "O"])
export const Coordinate = S.Literals([0, 1, 2])
export const Coordinates = S.Tuple([Coordinate, Coordinate])

export class OutOfTurnError extends S.TaggedErrorClass<OutOfTurnError>()("OutOfTurnError", {}) {}
export class SlotTakenError extends S.TaggedErrorClass<SlotTakenError>()("SlotTakenError", {}) {}

export class TicTacToeClient extends Client.Service<TicTacToeClient>()("examples/TicTacToeClient", {
  methods: {
    Move: {
      payload: S.Struct({
        position: Coordinates,
      }),
      success: S.Void,
      failure: S.Never,
    },
  },
  events: {
    GameStarted: {
      player: Player,
    },
    MoveMade: {
      player: Player,
      position: Coordinates,
    },
    GameEnded: {
      winner: S.optional(Player),
    },
  },
}) {}
```

## Define the actor

The actor ties a durable name to that client protocol and defines the per-client attachment state.

```ts
// TicTacToeActor.ts
import { Schema as S } from "effect"
import { Actor } from "liminal/actor"

import { Player, TicTacToeClient } from "./TicTacToeClient.ts"

export class TicTacToeActor extends Actor.Service<TicTacToeActor>()("examples/TicTacToeActor", {
  client: TicTacToeClient,
  name: S.String,
  attachments: { player: Player },
}) {}
```

The important split is:

- `name` is the unique identifier for the actor within its registry. Use `S.String` or a branded string schema.
- `attachments` are per-connected-client state.

For Tic-Tac-Toe, one actor represents one game, while attachments distinguish player `X` from player `O`.

## Send initial events from `onConnect`

`onConnect` is the lifecycle hook where you send a snapshot event to newly joined clients — or, as here, signal that the
game has started once both players are present.

```ts
import { Effect } from "effect"

import { TicTacToeActor } from "./TicTacToeActor.ts"

export const onConnect = Effect.gen(function* () {
  const { clients } = yield* TicTacToeActor
  const player = clients.size === 1 ? "X" : "O"
  yield* TicTacToeActor.all.send("GameStarted", { player }).pipe(Effect.orDie)
})
```

Pairing snapshot-on-connect with delta events keeps the UI story simple: connect, consume a snapshot, apply deltas — see
[Accumulator](../accumulator.mdx).

## Implement method handlers

Handlers run inside actor context.

```ts
// handleMove.ts
import { Effect } from "effect"

import { setBoard, getBoard } from "./Games.ts"
import { TicTacToeActor } from "./TicTacToeActor.ts"
import { OutOfTurnError, SlotTakenError } from "./TicTacToeClient.ts"

export const handleMove = TicTacToeActor.handler(
  "Move",
  Effect.fn(function* ({ position }) {
    const { currentClient, name: gameId } = yield* TicTacToeActor
    const { board, turn } = yield* getBoard(gameId)
    const { player } = yield* currentClient.attachments

    if (turn !== player) {
      return yield* new OutOfTurnError()
    }
    if (board[position[0]][position[1]] !== undefined) {
      return yield* new SlotTakenError()
    }

    yield* TicTacToeActor.all.send("MoveMade", { player, position })

    board[position[0]][position[1]] = player
    yield* setBoard({ board, turn: turn === "X" ? "O" : "X" })
  }),
)
```

The main server-side APIs used here are:

- `yield* TicTacToeActor` for actor context like `name`, `clients`, and `currentClient`
- `yield* client.attachments` to read per-socket state
- `yield* client.send(...)` for one-client events
- `yield* TicTacToeActor.all.send(...)` for broadcasts
- `yield* TicTacToeActor.others.send(...)` to broadcast to everyone except `currentClient`
- `yield* client.disconnect` or `yield* TicTacToeActor.all.disconnect` for disconnects

## Back the actor with durable state

Actors are not a database. For anything that must survive hibernation or eviction, persist state externally. The
tic-tac-toe example persists its board through Effect's `KeyValueStore`, backed by an R2 bucket via
`KeyValueStore.layerR2` from `liminal/platform`:

```ts
// KvLive.ts
import { Layer } from "effect"
import { R2 } from "liminal"
import { KeyValueStore } from "liminal/platform"

export const KvLive = KeyValueStore.layerR2().pipe(Layer.provide(R2.layer("BUCKET")))
```

Wrap your storage access in plain Effects that depend on `KeyValueStore`:

```ts
// Games.ts
import { Effect } from "effect"
import { KeyValueStore } from "effect/unstable/persistence"

export const getBoard = (gameId: string) =>
  Effect.gen(function* () {
    const kv = yield* KeyValueStore.KeyValueStore
    // read + decode the stored board
    // ...
  })

export const setBoard = (config: { readonly turn: "X" | "O"; readonly board: Board }) =>
  Effect.gen(function* () {
    const kv = yield* KeyValueStore.KeyValueStore
    // write the updated board
    // ...
  })
```

## Register the actor with `ActorRegistry`

`ActorRegistry` is the Cloudflare Durable Object wrapper around your actor.

```ts
// TicTacToeRegistry.ts
import { Layer } from "effect"
import { ActorRegistry } from "liminal"

import { handleMove } from "./handleMove.ts"
import { KvLive } from "./KvLive.ts"
import { onConnect } from "./onConnect.ts"
import { TicTacToeActor } from "./TicTacToeActor.ts"

export class TicTacToeRegistry extends ActorRegistry.Service<TicTacToeRegistry>()("examples/TicTacToeRegistry", {
  actor: TicTacToeActor,
  handlers: { Move: handleMove },
  onConnect,
  prelude: KvLive,
  runLayer: Layer.empty,
}) {}
```

The shape maps cleanly onto runtime concerns:

- `actor` is the Liminal actor definition.
- `prelude` is long-lived infrastructure (databases, config, asset bindings).
- `runLayer` is short-lived, request-local context derived from actor context.
- `handlers` is the method implementation table.
- `onConnect` runs for every newly upgraded socket.

The Durable Object binding name is supplied later, when the registry's `.layer(...)` is applied.

## Upgrade from an HTTP route

An HTTP route picks the actor name and initial attachments, then calls `upgrade(...)`.

```ts
// main.ts
import { Effect, Layer } from "effect"
import { HttpRouter, HttpServer, HttpServerResponse } from "effect/unstable/http"
import { Assets, Worker } from "liminal"

import { TicTacToeRegistry } from "./TicTacToeRegistry.ts"
import * as GameState from "./Games.ts"
import { KvLive } from "./KvLive.ts"

export { TicTacToeRegistry }

const ApiLive = Layer.mergeAll(
  HttpRouter.add("GET", "/", Effect.succeed(HttpServerResponse.text("ok"))),
  HttpRouter.add(
    "GET",
    "/play",
    Effect.gen(function* () {
      const { gameId, player } = yield* GameState.init
      return yield* TicTacToeRegistry.upgrade(gameId, { player })
    }).pipe(Effect.orDie),
  ),
  HttpRouter.cors({
    allowedHeaders: ["*"],
    allowedMethods: ["*"],
    allowedOrigins: ["*"],
  }),
  HttpRouter.add("*", "/*", Assets.forward),
)

export default Worker.make({
  handler: ApiLive.pipe(
    Layer.provide(HttpServer.layerServices),
    HttpRouter.toHttpEffect,
    Effect.flatMap((v) => v),
    Effect.provide(Layer.mergeAll(KvLive, TicTacToeRegistry.layer("TICTACTOE_REGISTRY"), Assets.layer("ASSETS"))),
    Effect.catchCause(() => Effect.succeed(HttpServerResponse.empty({ status: 500 }))),
  ),
  prelude: Layer.empty,
})
```

A few things worth pointing out:

- `HttpRouter.add(method, path, effect)` mounts a single route.
- `HttpRouter.cors(...)` wires up CORS for every route in the router.
- `HttpRouter.add("*", "/*", Assets.forward)` falls through to the Cloudflare Assets binding for unmatched requests.
- `Worker.make({ handler, prelude })` produces the Worker `fetch` export; combined with the Durable Object class
  re-export, it is everything your Worker entrypoint needs.
- Each binding is supplied as a layer by calling `.layer("ENV_NAME")` with the matching Wrangler binding name. For the
  registry, `TicTacToeRegistry.layer("TICTACTOE_REGISTRY")` points at the Durable Object binding.
- Every request that upgrades with the same actor name lands in the same Durable Object instance.

## Connect from the client

`Client.layerSocket(...)` turns the protocol into a live service over WebSocket.

```ts
import { BrowserSocket } from "@effect/platform-browser"
import { Layer, Stream, Effect } from "effect"
import { Client } from "liminal/actor"

import { TicTacToeClient } from "./TicTacToeClient.ts"

const TicTacToeClientLive = Client.layerSocket({
  client: TicTacToeClient,
  url: "/play",
  replay: { mode: "startup" },
}).pipe(Layer.provide(BrowserSocket.layerWebSocketConstructor))

Effect.gen(function* () {
  yield* TicTacToeClient.events.pipe(
    Stream.runForEach(
      Effect.fn(function* (event) {
        switch (event._tag) {
          case "GameStarted":
            break
          case "MoveMade":
            break
          case "GameEnded":
            break
        }
      }),
    ),
    Effect.forkScoped,
  )

  yield* TicTacToeClient.f("Move")({ position: [2, 1] })
}).pipe(Effect.provide(TicTacToeClientLive))
```
