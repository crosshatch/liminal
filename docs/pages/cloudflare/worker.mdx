# Worker

`Worker.make(...)` produces the `fetch` export for a Cloudflare Worker.

It does three things:

- builds a `ManagedRuntime` from your `prelude` layer and the Cloudflare env-backed `ConfigProvider`
- runs each request through your `handler` Effect
- provides `NativeRequest`, `HttpServerRequest`, `HttpClient`, and `ExecutionContext` to that handler

## Build the entrypoint

```ts
import { Layer, Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"
import { Assets, Worker } from "liminal"

export default Worker.make({
  handler: Assets.forward.pipe(
    Effect.provide(Assets.layer("ASSETS")),
    Effect.catch(() => Effect.succeed(HttpServerResponse.empty({ status: 500 }))),
  ),
  prelude: Layer.empty,
})
```

The shape is:

- `handler` — an `Effect<HttpServerResponse, HttpServerError, ...>` with access to the services below
- `prelude` — long-lived dependencies built once per worker instance (database clients, config, etc.)

Inside `handler`, these services are always available:

- `NativeRequest` — the raw `Request` (used by `Assets.forward` and any binding that expects a `Request`)
- `HttpServerRequest.HttpServerRequest` — the Effect Platform request wrapper
- `HttpClient.HttpClient` — `FetchHttpClient.layer` for outbound calls
- `ExecutionContext` — the Cloudflare execution context, useful for `waitUntil`

Configuration is wired through `ConfigProvider.fromUnknown(env)` so `Config.string("FOO")` resolves against the
Cloudflare env automatically.

## Composing with `HttpRouter`

`Worker.make(...)` accepts a single `Effect`. Build it from layers using `HttpRouter`:

```ts
import { Effect, Layer } from "effect"
import { HttpRouter, HttpServer, HttpServerResponse } from "effect/unstable/http"
import { Assets, Worker } from "liminal"

const ApiLive = Layer.mergeAll(
  HttpRouter.add("GET", "/", Effect.succeed(HttpServerResponse.text("ok"))),
  HttpRouter.add("*", "/*", Assets.forward),
)

export default Worker.make({
  handler: ApiLive.pipe(
    Layer.provide(HttpServer.layerServices),
    HttpRouter.toHttpEffect,
    Effect.flatMap((handler) => handler),
    Effect.provide(Assets.layer("ASSETS")),
    Effect.catchCause(() => Effect.succeed(HttpServerResponse.empty({ status: 500 }))),
  ),
  prelude: Layer.empty,
})
```

## Combining with `ActorRegistry`

When you have actors, also re-export each registry class so Cloudflare can instantiate the Durable Object:

```ts
export { ChatRegistry, LobbyRegistry } from "./registries.ts"
```

See [Registry and Routing](./registry-and-routing.mdx) for the full pattern.
